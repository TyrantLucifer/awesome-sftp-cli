# 预览、编辑、打开和搜索

[English](../../user/preview-edit-search.md)

AMSFTP 可以在不完整下载的情况下查看远端文件，为其他程序准备一份经过验证的本地
副本，也可以通过标准 SFTP 搜索远端目录树。这些流程共用受管理的本地缓存，但没有
任何一个会静默上传发生变化的文件。

## 预览文件

选中文件后按 `K`；也可以在文件上按 `l`。如果选中的是目录，`l` 仍然会正常
进入目录。

在预览中：

| 按键 | 作用 |
| --- | --- |
| `j` / `k` | 向下 / 向上滚动一行渲染内容 |
| `h` / `l` | 读取头部或尾部范围 |
| `gg` / `G` | 跳到头部或尾部范围 |
| `r` | 在自动、可用时的原始 JSON 和元数据视图之间切换 |
| `Esc` | 把焦点还给文件栏 |
| `K` | 预览获得焦点时将它关闭 |

预览会按较小的范围读取，并明确标出不完整内容。大文本文件不会为了预览而全部载入。
二进制数据和不安全的终端控制字节会经过清理，或者以元数据/十六进制显示，不会直接
写到终端。

只有终端明确证明支持兼容协议时才会显示图片。不支持、损坏或过大的图片会退回元数据
视图，不会发送猜测出来的转义序列。

预览符号链接时只显示元数据，不会沿链接打开目标。

## 使用终端编辑器编辑

1. 选中普通文件并按 `e`。
2. 检查解析后的编辑器命令并确认。
3. 编辑本地副本，然后退出编辑器。
4. 检查本地和远端各自发生了什么变化。
5. 选择上传、另存为、跳过、刷新或保留以便恢复。
6. 如果选择写入，再确认最终同步步骤。

编辑器按以下顺序选择：

1. AMSFTP 配置中的 `external.editor`；
2. `VISUAL`；
3. `EDITOR`；
4. 从 `PATH` 中找到的 `nvim`、`vim`、`vi`。

文件路径会作为一个独立的最后参数传入。AMSFTP 不会把文件名拼进 shell 命令。

编辑器退出后，AMSFTP 会把本地副本和远端文件分别与原始版本比较。常见结果如下：

| 本地文件 | 远端文件 | AMSFTP 提供的选择 |
| --- | --- | --- |
| 未变化 | 未变化 | 结束，不创建 Job |
| 已变化 | 未变化 | 上传、另存为、跳过或保留 |
| 未变化 | 已变化 | 刷新；不会推断为上传 |
| 已变化 | 已变化/删除/替换 | 查看冲突，明确覆盖、另存为、跳过或保留 |
| 无法验证 | 任意状态 | 不上传；明确保留或放弃 |

选择覆盖时，AMSFTP 会先把观察到的远端原文件保存在同目录中一个隐藏的 Job 专属
名称下，再发布编辑后的版本。它不会自动删除这份安全副本。如果远端再次变化，会
再次产生冲突，而不是直接覆盖。

上传中断时，未同步的本地副本和基线会留给恢复流程。按大写 `E` 打开编辑恢复，
使用 `j`/`k` 选择会话，按 `K` 检查，再按 `Enter` 检查或继续。

## 使用默认应用打开

按 `o` 可以准备本地副本，并使用平台默认程序打开：

- macOS：`/usr/bin/open`；
- Linux：`/usr/bin/xdg-open`。

可以通过 `external.opener` 替换默认程序。

桌面 opener 经常会在应用真正关闭前就返回，因此 AMSFTP 不会把进程退出或修改时间
变化当成上传许可。准备好检查变化时按 `Enter`，或者按 `Esc` 把会话留给以后
恢复。任何回写都使用与终端编辑相同的比较、确认和 Job 流程。

## 查找条目或搜索目录树

AMSFTP 提供三种不同的搜索：

| 按键 | 范围 |
| --- | --- |
| `/` | 在当前目录已经加载的条目中模糊跳转 |
| `f` | 在当前目录下递归搜索文件名 |
| `g/` | 在当前目录下递归搜索文件内容 |

文件名和内容搜索结果会持续进入搜索视图。可以使用普通文件动作打开、预览或复制
结果。

标准 SFTP 内容搜索需要通过连接读取文件片段，因此开始前 AMSFTP 会显示提示并要求
确认。默认会跳过二进制文件。

每次递归搜索都有深度、条目数、文件数、字节数、时长和结果数限制。达到限制时会返回
已经找到的内容，并明确标记为部分结果。取消搜索时也会保留已经显示的结果。搜索结果
只是某个时刻的快照；打开或修改前还会重新检查当前文件。

公开构建不分发可选的远端搜索加速组件。文件名和内容搜索仍然可以通过受限的标准
SFTP 通道工作。

## 选择缓存策略

状态栏会显示当前工作区的一种策略：

- **自动**（`lru`）：默认策略；旧的干净内容可以被清理。
- **临时**（`ephemeral`）：内容会更早进入可清理状态。
- **离线**（`pinned_offline`）：新准备的内容不会被常规清理，并且端点不可用时
  可能仍可打开。

当 `g` 路径输入框为空时：

| 组合键 | 作用 |
| --- | --- |
| `gp` | 在自动 → 临时 → 离线之间切换 |
| `gc` | 检查并清理当前工作区中符合条件的内容 |
| `gC` | 检查并清理所有工作区中符合条件的内容 |

按 `S` 可以把当前策略和工作区一起保存。

离线内容只是一份保留副本，并不能证明远端文件没有变化。在无法重新检查端点时，
AMSFTP 不会允许上传。如果有多个远端版本都可能匹配，它会拒绝猜测。

缓存默认全局上限为 2 GiB，每个工作区 1 GiB。正在用于预览、编辑、打开、上传或
恢复的文件不会被常规清理。清理只会删除受管理对象，绝不会删除远端文件或原始本地
文件。

## 配置编辑器或 opener

在 `config.json` 中使用结构化命令：

```json
{
  "schema_version": 1,
  "external": {
    "editor": {
      "executable": "nvim",
      "argv": ["-f"]
    },
    "opener": {
      "executable": "/absolute/path/to/opener",
      "argv": []
    }
  }
}
```

AMSFTP 会直接运行这些程序，不经过 shell。准备好的文件路径会自动添加为最后一个
参数，因此不要再写占位符或 shell 引号。

编辑器、opener 和外部预览器都使用当前用户权限运行，可以读取该程序通常有权读取
的内容。AMSFTP 不会向它传递远端凭据或 daemon socket。

路径、验证方式和外部预览规则见[配置](configuration.md)。

## 一次性命令和交互 shell

按 `!` 输入一条命令。AMSFTP 会展示端点、工作目录和命令文本，要求第二次确认后
才运行。命令运行时按 `Esc` 可以取消本地进程组或 SSH 会话。

按 `gs` 在当前目录打开交互 shell。远端栏无法建立所选工作目录时，使用 `gS`
可以明确回退到该账号的 home 目录。

这些命令使用新的进程或 SSH 连接。远端执行会关闭 Agent forwarding、凭据委托和
持久连接共享。交互终端字节和一次性命令输出都不会持久保存。

与文件动作不同，shell 命令不会包装成传输 Job。请像检查普通 shell 命令一样认真
检查它们。
