# 日常使用

[English](../../user/everyday-use.md)

AMSFTP 会一直显示两个互相独立的文件栏。高亮的一栏是活动栏；除非某个动作明确把
另一栏作为目标，导航和命令都作用于活动栏。

## 打开和切换位置

启动时可以不带位置，也可以带一个或两个位置：

```sh
amsftp
amsftp /absolute/local/path
amsftp /left/path work:/remote/path
amsftp host-a:/path/a host-b:/path/b
```

按 `Tab` 切换文件栏。按 `c` 只为当前栏选择新端点。端点选择器包含 `local` 和
明确的 OpenSSH 别名；输入文字可以过滤，使用方向键选择，按 `Enter` 确认。

从用户视角看，端点切换是一次完整操作：只有新端点成功连接并返回第一页目录内容，
旧文件栏才会被替换，否则它仍然可用。

## 浏览目录

| 按键 | 作用 |
| --- | --- |
| `j` / `k`、`↓` / `↑` | 下移或上移 |
| `h`、`←` | 打开上级目录 |
| `l`、`→` | 进入目录或预览所选文件 |
| `g` | 为当前栏输入绝对路径 |
| `/` | 在当前目录已加载的条目中模糊查找 |
| `s` | 切换排序方式 |
| `H` | 显示或隐藏隐藏条目 |
| `R` | 刷新当前目录 |
| `Tab` | 切换活动栏 |

使用 `/` 时，输入查询并用 Up/Down 选择匹配项。按 `Enter` 后，光标会定位到完整
列表中的对应条目；按 `Esc` 会回到搜索前的位置。这只是当前目录内的快速跳转，不是
递归搜索。

安全的导航和选择动作支持数字前缀，例如 `12j` 会向下移动十二项。数字不会取消
破坏性操作原本需要的确认。

目录读取和界面渲染采用分页。即使目录非常大，也不会为了显示它而一次加载并绘制
全部条目。

## 选择文件

- `Space` 切换单个离散标记。
- `v` 开始或结束 Visual 选择。
- `V` 开始或结束 Visual Line 选择。
- `Esc` 退出当前选择，或关闭最内层提示。

没有选择时，文件操作使用光标所在条目。选择多个条目后，动作栏只会显示适用于整个
选择的操作。

`y` 记录复制，`d` 记录移动，`p` 粘贴到另一栏。小写 `d` 永远不会删除文件。
大写 `D` 才是独立且需要确认的删除动作。

移动或删除重要数据前，请先阅读[传输和 Job](transfers.md)。

## 使用动作栏和抽屉

状态栏上方的动作栏会根据当前对象和实际键位变化。想知道下一步能做什么时，通常看
这里最快。

| 按键 | 抽屉 |
| --- | --- |
| `K` | 预览 |
| `J` | 后台 Job |
| `L` | 已脱敏的诊断日志 |

抽屉获得焦点后，再按一次对应按键会将它关闭。按 `Esc` 会把焦点还给文件栏，但保留
抽屉。底部同一时间只会激活一个抽屉。

Job 抽屉会适应终端宽度。宽终端会在列表旁显示所选 Job 的详情；窄终端使用紧凑的
路径摘要。标题只会显示当前 Job 状态真正允许的控制动作。

## 保存工作区

按 `S` 可以保存当前双栏布局。工作区会记住位置、活动栏、排序与过滤选项、隐藏文件
可见性和缓存策略。

以后这样打开：

```sh
amsftp --workspace <name>
```

启动选择器中也会列出保存过的工作区。如果工作区文件损坏，AMSFTP 会保留原文件并
回到选择器，不会直接覆盖它。

远端位置只保存 OpenSSH 别名，不保存凭据或展开后的 SSH 配置。

## 重连和刷新

每一栏都有自己的连接。如果某个 SSH 服务器离线，对应栏会报告问题，并且只重试
可以安全重试的错误；另一栏仍然可用。

重连后，AMSFTP 会确认原目录是否仍然存在。如有必要，它会打开最近的可访问上级
目录，并提示发生了回退。需要明确重新读取时，随时可以按 `R`。

如果本地 daemon 重启，TUI 会等待私有实例完成交接，重新连接两栏，并忽略旧连接
迟到的结果。持久 Job 的恢复与屏幕状态分开处理。

## 运行命令或 shell

- `!` 在当前栏目录中运行一条经过确认的命令。
- `gs` 在该目录打开交互 shell。
- 远端工作目录无法建立时，`gS` 明确改为打开远端账号的 home shell。

一次性命令运行前，AMSFTP 会展示确认。远端命令使用新的 OpenSSH 连接，不会开启
Agent forwarding 或凭据委托。shell 输出不会写入持久日志。

这些能力等同于正常 shell 访问：命令可以使用当前账号权限改动文件，并不受 AMSFTP
传输确认流程保护。

## 退出 AMSFTP

按 `q` 或 `Ctrl-C` 会恢复终端并退出客户端，但不会取消后台 Job。确实需要暂停、
恢复或取消任务时，请使用 Job 抽屉或 `amsftp job` 命令。

终端最小尺寸是 20 列 × 5 行；更大的终端能更清楚地显示双栏布局。
